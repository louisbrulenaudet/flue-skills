## Source directory selection

Flue discovers your application from one source directory, chosen in priority order:

1. **`.flue/`** ‚Äî use when embedding Flue inside a larger existing application.
2. **`src/`** ‚Äî **recommended for new projects.**
3. **Project root** ‚Äî compact layout for very small projects.

The build output directory defaults to `dist/` and is configurable in `flue.config.ts` via the `output` property.

## What lives where

- **`flue.config.ts`** ‚Äî generated by `flue init --target <node|cloudflare>`. Holds target and `output`.
- **`src/app.ts`** ‚Äî optional. Export a Hono app to compose your own routes/middleware with `flue()`. Without it, Flue mounts its routes at `/` automatically. See `routing.md`.
- **`src/cloudflare.ts`** ‚Äî optional custom Worker exports for Cloudflare.
- **`src/db.ts`** ‚Äî optional (Node targets). Exports a `PersistenceAdapter` (`sqlite(...)` or `postgres(...)`) for Flue's own store. Cloudflare ignores this ‚Äî it uses Durable Object SQLite automatically. See `database.md`.
- **`src/agents/*.ts`** ‚Äî one agent per file; the filename is the agent name.
- **`src/workflows/*.ts`** ‚Äî one workflow per file; the filename is the workflow name.
- **`src/channels/*.ts`** ‚Äî provider ingress modules (often scaffolded by `flue add channel <name>`).
- **`src/skills/<name>/SKILL.md`** ‚Äî reusable instructions with optional `references/`.
- **Shared code / tools** ‚Äî put plain modules (e.g. `src/tools/*.ts`, `src/lib/*.ts`) outside the discovered `agents/`/`workflows/`/`channels/` folders and import them. Tools are not auto-discovered; they are imported into agent/workflow config.

## Naming conventions

- Agent and workflow **filenames** are **lower-kebab-case** (`support-assistant.ts`, `summarize-ticket.ts`). This keeps names portable across deployment targets and determines the HTTP route segment and the generated Cloudflare Durable Object class name (`FlueSupportAssistantAgent`, `FlueSummarizeTicketWorkflow`).
- Skill `name` frontmatter is kebab-case, ‚â§ 64 chars, no leading/trailing/consecutive hyphens.

## Discovery rules

`agents/`, `workflows/`, and `channels/` are discovered from **flat (non-nested) files** in the source directory. Do not nest agent/workflow files in subfolders expecting discovery. Skills are the exception: each skill is a folder containing `SKILL.md`.

## Anti-patterns

- Mixing two source roots (`.flue/` and `src/`) ‚Äî Flue picks one by priority; the other is ignored and confuses readers.
- Nesting agents/workflows in subdirectories and expecting auto-discovery.
- Putting shared tools inside `agents/` (they'd be misread as agents). Keep them in a non-discovered folder and import.
- Editing the generated `dist/` output by hand.
